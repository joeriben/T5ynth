# T5ynth-Presets — repo bootstrap

Drop these files into the `joeriben/T5ynth-Presets` GitHub repository (or
any fork of it) to enable in-plugin "Update from UCDCAE AI Lab".

## Layout expected by the plugin

```
T5ynth-Presets/
├── manifest.json                  ← generated by CI (do not edit)
├── Analog Saw C2C3.t5p            ← .t5p files live at the repo root
├── Heaven and Hell WT.t5p
├── ...
├── scripts/
│   └── generate_manifest.py       ← run by the Action
└── .github/workflows/manifest.yml ← triggers on every push to a .t5p
```

## What CI does

`manifest.yml` fires on push to any `*.t5p` at the repo root. It runs
`generate_manifest.py`, which scans every `.t5p`, reads its embedded JSON
header for the "name" field, computes SHA256, and writes `manifest.json`
at the repo root. If the manifest changed, CI commits it back. The path
filter only matches `*.t5p` — the manifest commit itself doesn't
re-trigger the workflow.

## Your workflow as maintainer

```
cp ~/MyNewPreset.t5p .
git add MyNewPreset.t5p
git commit -m "add: MyNewPreset"
git push
# ~30 s later: manifest.json is regenerated by Actions
# next user click on "Update Library" pulls the new preset
```

To remove a preset upstream: `git rm Foo.t5p && git push`. The plugin's
pruning pass deletes any local file no longer in the manifest, so users
see the removal on their next update.

## Manual generation (no CI)

```
python scripts/generate_manifest.py
git add manifest.json && git commit -m "regenerate manifest"
```

## Manifest schema (v1)

```json
{
  "schema_version": 1,
  "generated_at": "2026-05-17T12:00:00Z",
  "base_url": "https://raw.githubusercontent.com/joeriben/T5ynth-Presets/main/",
  "presets": [
    {
      "name": "Heaven and Hell WT",
      "path": "Heaven and Hell WT.t5p",
      "size": 4096324,
      "sha256": "deadbeef..."
    }
  ]
}
```

The plugin keys deduplication on `path` and refresh decisions on `sha256`.
